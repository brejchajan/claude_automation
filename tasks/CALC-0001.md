## Description

Implement a simple command line calculator in Python. The calculator runs as an
interactive REPL (read-eval-print loop). On startup it prints a welcome message
and a prompt. The user types a mathematical expression (numbers and operators
such as +, -, *, /, **, parentheses), presses Enter, and the result is printed
to the screen. The calculator keeps accepting new expressions until the user
types "exit" or "quit" (case-insensitive), or sends EOF (Ctrl+D).

The implementation must live in a single file `calculator.py` at the project
root. It must be runnable with `python calculator.py`.

Use Python's `ast.literal_eval` or a safe expression parser -- do NOT use
`eval()` or `exec()` on raw user input. A good approach is the `ast` module:
parse the expression, walk the AST, and only allow numeric literals and the
arithmetic operators (+, -, *, /, //, %, **). Reject anything else with a
friendly error message.

## Acceptance Criteria

- Running `python calculator.py` starts an interactive prompt.
- Entering `2 + 3` prints `5`.
- Entering `10 / 3` prints `3.3333333333333335` (or similar float representation).
- Entering `2 ** 10` prints `1024`.
- Entering `(1 + 2) * (3 + 4)` prints `21`.
- Entering invalid input like `hello` prints a user-friendly error, not a traceback.
- Typing `exit` or `quit` ends the program gracefully.
- Pressing Ctrl+D (EOF) ends the program gracefully.
- The calculator never calls `eval()` or `exec()` on user input.
- Unit tests exist in `test_calculator.py` covering basic arithmetic, operator
  precedence, parentheses, division by zero, and invalid input handling.
